OVERVIEW

The Real-Time Weather Data Pipeline is an end-to-end data engineering solution that demonstrates how to collect, process, store, and visualize live weather information in real time. It integrates multiple technologies including Apache Kafka, Spark Streaming, PostgreSQL, and Streamlit to create a robust and scalable streaming architecture. Data is fetched from an external weather API, processed using Spark, stored in a relational database, and displayed through an interactive dashboard. All components are containerized using Docker Compose, allowing seamless deployment and orchestration of services in a production-like environment.

KEY ACHIEVEMENTS

  • Built a real-time data pipeline that collects and processes weather data from 50+ cities, integrating Kafka, Spark Streaming, and PostgreSQL for seamless end-to-end flow.
  • Achieved low-latency ingestion and transformation, maintaining an end-to-end data flow time under 5 seconds from API fetch to database insertion.
  • Developed a responsive Streamlit dashboard that automatically refreshes every 10 seconds, allowing users to view real-time weather metrics such as temperature, humidity, and alerts.
  • Containerized and deployed 6+ integrated services (Kafka, Spark, PostgreSQL, pgAdmin, Streamlit, Zookeeper) using Docker Compose, enabling full system startup in under 2 minutes.
